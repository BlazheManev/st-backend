Data Model Documentation

User Entity

The user entity represents individuals who can log in and out of the system. Each user has the following attributes:

  • ime: The first name of the user.
  • priimek: The surname of the user.
  • email: The email address of the user, used for login and communication.
  • dan: An array of objects representing days, with each object containing:
    • datum: The date of the entry/exit.
    • vhodi: An array of strings, each representing a timestamp when the user entered.
    • izhodi: An array of strings, each representing a timestamp when the user exited.
  • hash: A hash of the user's password for security.
  • salt: A unique salt for hashing the password, enhancing security.

Data Availability and Synchronization

On-Device Data: The application will always store the user's basic information (name, email) and the hash/salt for authentication on the device.

Server-Only Data: Detailed logs of user entries and exits are stored only on the server to maintain data integrity and prevent tampering.

Synchronization: Data synchronization occurs in real-time for login and entry/exit recording. Full synchronization of logs is done daily to ensure the user's device is up-to-date with all recent activities.
